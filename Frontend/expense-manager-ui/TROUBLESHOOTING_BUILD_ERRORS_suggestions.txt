7) TS2349: This expression is not callable — subscribe argument typing issue

Errors:
  TS2349 in:
    - src/app/features/categories.component/categories.component.ts:72:8
    - src/app/features/transactions.component/transactions.component.ts:139:8

What’s likely wrong:
  - In CategoriesComponent:
      const obs = this.editingId() ? update(...) : create(...)
      obs.subscribe({ ... })
    TypeScript infers obs as a union of two different observable types (or function overloads),
    and the union’s subscribe signature becomes incompatible.

Suggested non-invasive solutions:
  - Explicitly type obs as Observable<unknown> (or Observable<SomeDto>) before calling subscribe.
  - Or avoid union by splitting into if/else and calling subscribe in each branch.
  - Or use firstValueFrom(...) with await in a dedicated handler.

8) NG5002: Only void, custom and foreign elements can be self closed "canvas"

Errors:
  NG5002 in:
    - src/app/features/dashboard.component/dashboard.component.html
      <canvas ... /> self-closed

What’s likely wrong:
  - In Angular templates, HTMLCanvasElement cannot be self-closed with />.
  - Must be written as opening+closing tags: <canvas>...</canvas> or just <canvas ...></canvas>.
  - With directives like baseChart, you should not self-close the canvas.

Suggested fix:
  - Change:
      <canvas baseChart ... />
    to:
      <canvas baseChart ...></canvas>
